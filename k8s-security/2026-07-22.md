## 🔴 Critical

### 1. Unpatched Argo CD Repo-Server Flaw — Cluster Takeover (No CVE)
**Source:** The Hacker News (Jul 01, 2026)
An unpatched vulnerability in the Argo CD repo-server component allows an **unauthenticated attacker** to execute arbitrary code if they can reach the component's internal network port. Synacktiv researchers demonstrated that this leads to **full Kubernetes cluster takeover**. There is currently **no fix and no CVE assigned** — the flaw was reported to Argo CD's maintainers but remains unpatched.
- **Mitigation:** Restrict network access to the repo-server port; monitor for unexpected connections to port 8081 (gRPC) or 8080 (HTTP).
- **Reference:** https://thehackernews.com/2026/07/unpatched-argo-cd-repo-server-flaw.html

### 2. TeamPCP / PCPJack Credential Stealer — Worm-Like Cloud Spread
**Source:** The Hacker News (Mar 24, 2026)
A credential theft framework dubbed **PCPJack** targets exposed cloud infrastructure with worm-like propagation. Harvests credentials from cloud, container, developer, productivity, and financial services, exfiltrating through attacker-controlled infrastructure while removing TeamPCP artifacts from compromised environments. The toolset exploits **5 CVEs** to spread autonomously across cloud systems.
- **Action:** Audit exposed cloud services; enforce network segmentation; review credential rotation policies.

### 3. Trivy Docker Hub Supply Chain Attack — Infostealer + K8s Wiper
**Source:** The Hacker News (Feb 09, 2026)
The **Trivy supply chain compromise** on Docker Hub continues to have cascading impact. Malicious versions 0.69.4, 0.69.5, and 0.69.6 delivered infostealer malware with worm capabilities and a **Kubernetes wiper** component. The last known clean release is 0.69.3. New image tags have since been removed from Docker Hub, but the blast radius across developer environments is significant.
- **Action:** Verify Trivy deployment versions; check for usage of affected tags in CI/CD and runtime.

## 🟡 High / Medium

### 4. CVE-2026-4342 — ingress-nginx Comment-Based Config Injection
**Source:** K8s Security Announce (Mar 20, 2026) | **CVSS 8.8 (High)**
A combination of Ingress annotations can be used to inject configuration into nginx, leading to **arbitrary code execution** in the ingress-nginx controller and **disclosure of cluster-wide Secrets** (the controller can access all Secrets by default).
- **Vector:** `CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H`
- **Fixed in:** ingress-nginx v1.15.1, v1.14.5, v1.13.9
- **Reference:** https://groups.google.com/g/kubernetes-security-announce/c/hIAkgZb8MJ0

### 5. CVE-2026-3288 — ingress-nginx Rewrite-Target Config Injection
**Source:** K8s Security Announce (Mar 10, 2026) | **CVSS 8.8 (High)**
The `nginx.ingress.kubernetes.io/rewrite-target` annotation can be used to inject nginx configuration, leading to arbitrary code execution and Secret disclosure — same impact pattern as CVE-2026-4342.
- **Vector:** `CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H`
- **Mitigation (pre-upgrade):** Use admission control to block the `rewrite-target` annotation.
- **Reference:** https://groups.google.com/g/kubernetes-security-announce/c/E9bLHAD6-eg

### 6. Multiple ingress-nginx CVEs (Feb 2026 Batch) — High 8.8

### 7. CVE-2026-3865 — CSI Driver for SMB Path Traversal (subDir)
**Source:** K8s Security Announce (Apr 11, 2026) | **CVSS 6.5 (Medium)**
Insufficient validation of the `subDir` parameter in SMB CSI volume identifiers allows path traversal. A user with PersistentVolume creation privileges can craft a `volumeHandle` with `../` sequences to delete/modify directories outside the intended managed subdirectory on the SMB server.
- **Vector:** `CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:H`
- **Fixed in:** CSI Driver for SMB v1.20.1+
- **Reported by:** Shaul Ben Hai (SentinelOne)
- **Reference:** https://groups.google.com/g/kubernetes-security-announce/c/QNv6lg4RxWE

### 8. CVE-2026-3864 — CSI Driver for NFS Path Traversal (subDir)
**Source:** K8s Security Announce (Mar 17, 2026) | **CVSS 6.5 (Medium)**
Identical vulnerability pattern to CVE-2026-3865 but affecting the **NFS CSI driver**. Path traversal via `subDir` parameter allows unintended directory deletion/modification on the NFS server.
- **Reference:** https://groups.google.com/g/kubernetes-security-announce/c/i4ZKN9VLcUE

### 9. CVE-2026-34040 — Docker Engine AuthZ Bypass
**Source:** The Hacker News (Mar 24, 2026) | **CVSS 8.8 (High)**
Allows an attacker to bypass authorization plugins (AuthZ) under specific circumstances. Stems from an **incomplete fix for CVE-2024-41110** (a maximum-severity vulnerability in the same component from July 2024). Relevant for any containerized Kubernetes node workloads running Docker.
- **Reference:** https://thehackernews.com/2026/03/teampcp-hacks-checkmarx-github-actions.html

### 10. CVE-2025-14269 — Headlamp Credential Caching (Helm-Enabled)

## 📈 Trends & Recommendations

1. **Ingress-nginx is a primary attack surface**: 8 CVEs in 2026 (compared to ~2-3 for core K8s). The retirement announcement and ongoing CVE wave make migration to Gateway API / ingress2gateway the single highest-impact security action for most teams.

2. **CSI driver path traversal pattern**: Two nearly identical vulnerabilities (SMB + NFS) within one month reveal a pattern — CSI drivers with path-based parameters (subDir, volumeHandle) are an emerging vulnerability class. Audit any CSI driver that accepts path traversal sequences.

3. **TeamPCP supply chain attacks escalating**: The Trivy Docker Hub compromise (Feb 2026) followed by PCPJack (Mar 2026) show an organized campaign targeting cloud/container tooling supply chains. Verify container image provenance and pin versions to known-good hashes.

4. **Argo CD repo-server exposure**: The unpatched Argo CD flaw with no CVE or fix in sight is concerning for any team running Argo CD. Network segmentation of the repo-server is the only current mitigation.

5. **Ingress-nginx retirement urgency**: With EOL passed (March 2026) and no more security patches for older versions, teams still on ingress-nginx are operating without a safety net. The `ingress2gateway` tool (v1.0 released Mar 2026) provides a migration path.
