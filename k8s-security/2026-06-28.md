# Kubernetes Security Briefing — 2026-06-28

## Latest CVEs

### 🔴 Critical / High Severity

**CVE-2026-4342 — ingress-nginx Comment-Based Configuration Injection**
- Severity: **High (CVSS 8.8)**
- Date: March 20, 2026
- Description: Combination of Ingress annotations can be used to inject configuration into nginx, leading to arbitrary code execution in the context of the ingress-nginx controller and disclosure of Secrets accessible cluster-wide.
- Affected: ingress-nginx < v1.13.9 / < v1.14.5 / < v1.15.1
- Fix: Upgrade to v1.13.9, v1.14.5, or v1.15.1+
- Detection: Suspicious data within `rules.http.paths.path` field of Ingress resources

**CVE-2026-3288 — ingress-nginx rewrite-target Injection**
- Severity: **High (CVSS 8.8)**
- Date: March 10, 2026
- Description: The `nginx.ingress.kubernetes.io/rewrite-target` annotation can be used to inject configuration into nginx, leading to arbitrary code execution and credential disclosure.
- Affected: ingress-nginx < v1.13.8 / < v1.14.4 / < v1.15.0
- Fix: Upgrade to v1.13.8, v1.14.4, or v1.15.0+
- Mitigation (pre-upgrade): Use admission control to block the rewrite-target annotation

**CVE-2026-1580, CVE-2026-24512, CVE-2026-24513, CVE-2026-24514 — Multiple ingress-nginx Issues**
- Severity: **High (CVSS 8.8)** (most serious)
- Date: February 2, 2026
- Description: Multiple issues disclosed in ingress-nginx; most serious rated HIGH at 8.8.
- Affected: ingress-nginx < v1.13.7 / < v1.14.3
- Fix: Upgrade to v1.13.7, v1.14.3, or later

### 🟡 Medium Severity

**CVE-2026-3865 — CSI Driver for SMB Path Traversal**
- Severity: **Medium (CVSS 6.5)**
- Date: April 11, 2026
- Description: Insufficient validation of the `subDir` parameter in volume identifiers allows path traversal. A malicious user creating a PersistentVolume referencing the SMB CSI driver could craft a `volumeHandle` containing `../` traversal sequences. During cleanup operations, this may cause the driver to delete/modify directories outside the managed subdirectory on the SMB server.
- Affected: CSI Driver for SMB < v1.20.1
- Fix: Upgrade to v1.20.1+
- Mitigation: Restrict PersistentVolume creation to trusted administrators

**CVE-2026-3864 — CSI Driver for NFS Path Traversal**
- Severity: **Medium (CVSS 6.5)**
- Date: March 17, 2026
- Description: Insufficient validation of the `subDir` parameter in volume identifiers allows path traversal, same pattern as CVE-2026-3865 but for NFS CSI driver.
- Affected: CSI Driver for NFS < v4.13.1
- Fix: Upgrade to v4.13.1+
- Mitigation: Restrict PersistentVolume creation to trusted administrators

## 🔮 Trends & Recommendations

### 1. TeamPCP Dominates the Threat Landscape
TeamPCP has orchestrated the most significant K8s supply chain attack wave in 2026, with cascading compromises:
> Trivy → KICS → LiteLLM → Checkmarx → GitHub

**Recommendation:** Audit your software supply chain for any dependencies on these compromised tools/versions. Monitor for malicious Docker images and PyPI packages. Implement SLSA levels and software attestation.

### 2. ingress-nginx Remains the Top Kubernetes CVE Vector
Four distinct 2026 advisories (CVE-2026-4342, CVE-2026-3288, plus the February multi-CVE bundle) all target ingress-nginx. The project's retirement was announced in January 2026 with end-of-life in March 2026.

**Recommendation:** **Migrate from ingress-nginx to Gateway API** using Ingress2Gateway. If remaining on ingress-nginx, ensure you're on at least v1.15.1 and block dangerous annotations via admission control.

### 3. CSI Drivers — New Attack Surface
The SMB and NFS CSI driver path traversal CVEs (CVE-2026-3865, CVE-2026-3864) highlight that storage drivers are a growing attack surface.

**Recommendation:** Restrict PersistentVolume creation to trusted administrators. Upgrade CSI drivers to latest versions. Validate volumeHandle fields for traversal sequences.

### 4. Unfixed Architectural CVEs Now Explicit
The CVE record corrections (CVE-2020-8561, CVE-2020-8562, CVE-2021-25740) mean vulnerability scanners will flag these issues post-June 1. These cannot be patched — only mitigated through configuration.

**Recommendation:** Implement the mitigations documented in the blog post:
- Disable dynamic profiling and restrict log verbosity
- Use local DNS caching (dnsmasq) for API server
- Harden RBAC for Endpoints/EndpointSlices

### 5. Worm-Like Credential Stealers on the Rise
PCPJack represents a new class of automated, worm-like credential theft targeting cloud-native infrastructure.

**Recommendation:** Ensure Docker/K8s APIs are not exposed to the internet. Implement network policies, enable audit logging, and use runtime security tools (Falco, etc.) to detect lateral movement.
