## 🔴 Critical

### 🔴 CVE-2026-33634 — Trivy Supply Chain Compromise (CVSS 9.4)
TeamPCP threat actor compromised Aqua Security's Trivy vulnerability scanner CI/CD pipeline, pushing trojanized versions (0.69.4–0.69.6) to Docker Hub. The malicious images contained a credential stealer targeting SSH keys, cloud credentials, Kubernetes secrets, cryptocurrency wallets, and .env files. The compromise cascaded into Checkmarx GitHub Actions via stolen CI credentials and later into LiteLLM (1.82.7–1.82.8) via Trivy CI/CD dependency. **Worm-like propagation across cloud infrastructure** with Kubernetes lateral movement toolkit deploying privileged pods to every node.

### 🔴 CVE-2026-4342 — ingress-nginx Comment-Based Config Injection (CVSS 8.8 — HIGH)
Ingress annotations can be combined to inject nginx configuration, leading to **arbitrary code execution** in the ingress-nginx controller context and disclosure of cluster-wide Secrets. Affects ingress-nginx with no workaround other than upgrading. *(Latest in a wave of ingress-nginx CVEs — see also CVE-2026-3288 below)*

### 🔴 CVE-2026-3288 — ingress-nginx rewrite-target Config Injection (CVSS 8.8 — HIGH)
The `nginx.ingress.kubernetes.io/rewrite-target` annotation allows nginx configuration injection, enabling arbitrary code execution in the ingress-nginx controller. Affects ingress-nginx. Upgrade required.

### 🔴 CVE-2026-34040 — Docker Engine AuthZ Bypass (CVSS 8.8 — HIGH)
Incomplete fix for CVE-2024-41110. Attackers can bypass authorization plugins by crafting API requests that the Docker daemon forwards without the request body. Affects anyone relying on AuthZ plugins that introspect request bodies.

## 🟡 High/Medium

### 🟡 CVE-2026-3865 — CSI Driver for SMB Path Traversal (Medium)
Insufficient validation of `subDir` parameter in SMB CSI driver volume identifiers allows path traversal. Malicious users with PersistentVolume creation ability can craft `volumeHandle` with `../` sequences to delete or modify unintended directories on the SMB server during volume cleanup.

### 🟡 CVE-2026-3864 — CSI Driver for NFS Path Traversal (Medium)
Same class of vulnerability as CVE-2026-3865 but affecting the NFS CSI driver. Path traversal via `subDir` parameter allows unintended directory operations on the NFS server during volume deletion.

### 🟡 Multiple ingress-nginx Issues (CVE-2026-1580, CVE-2026-24512, CVE-2026-24513, CVE-2026-24514) — HIGH (CVSS 8.8)
A batch of four additional ingress-nginx CVEs disclosed together. Most serious rated HIGH (CVSS 8.8). Affects ingress-nginx < v1.13.7 and < v1.14.3.

### 🟡 CVE-2025-14269 — Headlamp Credential Caching (CVSS 8.8 — HIGH)
Unauthenticated users can reuse cached credentials to access Helm functionality through the Headlamp UI if Headlamp is installed with `config.enableHelm: true` and an authorized user has previously accessed Helm.

### 🟡 CVE-2025-13281 — Portworx Half-Blind SSRF in kube-controller-manager (Medium)
Authorized users can leak arbitrary information from unprotected endpoints in the control plane's host network via the in-tree Portworx StorageClass. Disabled by default since v1.31 via CSIMigrationPortworx feature gate. Supported versions >= v1.32 are not impacted.

### 🟡 CVE-2025-7445 — secrets-store-sync-controller Token Disclosure (CVSS 6.5 — MEDIUM)
Service account tokens logged when there's a marshaling error, potentially allowing access to cloud vault secrets. Tokens only logged on specific error conditions.

### 🟡 CVE-2025-7342 — K8s Image Builder Default Credentials (Windows)
VM images built with Kubernetes Image Builder (Nutanix/OVA providers) use default credentials for Windows images if user did not override.

### 🟡 CVE-2025-5187 — Nodes Can Delete Themselves via OwnerReference
NodeRestriction admission controller doesn't prevent nodes from patching themselves with an OwnerReference to a cluster-scoped resource. If the OwnerReference resource doesn't exist or is deleted, the node object is deleted via garbage collection. Allows node-level DoS.

### 🟡 CVE-2025-4563 — Nodes Can Bypass Dynamic Resource Allocation Checks
Low CVSS rationale but allows nodes to bypass dynamic resource allocation authorization.

### 🟡 CVE-2025-9708 — C# Client Improper Certificate Validation (CVSS 6.8 — MEDIUM)
The Kubernetes C# client accepts certificates from any CA without verifying the trust chain, enabling man-in-the-middle attacks and API impersonation.

### 🟡 CVE-2025-0426 — Node DoS via kubelet Checkpoint API
Large number of checkpoint requests can cause node denial of service.

### 🟡 Critical NVIDIA Container Toolkit Flaw — CVE-2025-23266 (CVSS 9.0)
Container escape vulnerability in NVIDIA Container Toolkit. Impacts all versions up to 1.17.7 and GPU Operator up to 25.3.0.

### 🟡 New Fluent Bit Flaws (5 CVEs) — RCE and Cloud Infrastructure Takeover
Five vulnerabilities in Fluent Bit (CVE-2025-12972, CVE-2025-12970, etc.) allowing path traversal, RCE, DoS, and tag manipulation. Affects cloud and K8s infrastructure using Fluent Bit for telemetry.

### 🟡 Chaos Mesh Critical GraphQL Flaws (CVE-2025-59358, etc.) — "Chaotic Deputy"
Multiple CVSS 7.5 flaws enabling cluster takeover in Kubernetes environments. Attackers need minimal in-cluster network access. Includes unauthenticated GraphQL debugging server exposure.

## ℹ️ Informational / Low

### ℹ️ K8s Blog: "Reconciling the Past" — Unfixed CVEs Corrected (May 26, 2026)
The Kubernetes SRC corrected CVE records for **CVE-2020-8561**, **CVE-2020-8562**, **CVE-2021-25740**, and **CVE-2020-8554** to reflect they are **unfixed architectural issues**, not patchable vulnerabilities:
- **CVE-2020-8561** (Medium 4.1): Webhook redirect in kube-apiserver — mitigate by restricting log verbosity
- **CVE-2020-8562** (Low 3.1): Proxy bypass via DNS TOCTOU — mitigate with local DNS caching
- **CVE-2021-25740** (Low 3.1): Cross-namespace forwarding via Endpoints — mitigate with RBAC hardening
- **CVE-2020-8554**: Kubernetes Service external IPs — all versions affected

CVE records updated June 1, 2026. Vulnerability scanners may now flag these where they weren't detected before. No code fix exists — administrative mitigations only.

### ℹ️ K8s Blog: Ingress NGINX Retirement (EOL March 2026)
Ingress NGINX reached end-of-life in March 2026. The project was retired by the Kubernetes Steering and Security Response Committees. Users are encouraged to migrate to Gateway API alternatives. The ongoing wave of ingress-nginx CVEs (CVE-2026-4342, CVE-2026-3288, and 4 more) underscores the urgency of migration.

### ℹ️ K8s Blog: Dashboard to Headlamp Transition (June 1, 2026)
Kubernetes Dashboard is transitioning to Headlamp. CVE-2025-14269 (credential caching) context.
