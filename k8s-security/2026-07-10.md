## 🔴 Critical

### 1. Unpatched Argo CD Repo-Server RCE — Full Cluster Takeover
**Source:** The Hacker News (Jul 01, 2026)
**CVE:** None assigned — no fix available
**Severity:** 🔴 Critical (unauthenticated RCE → cluster takeover)

Argo CD's `repo-server` component contains an unpatched flaw where its internal gRPC service has **no authentication**. An attacker who can reach the gRPC port (default: no network policies in Helm installs) can send a crafted request to execute arbitrary commands via the `kustomize --helm-command` option.

**Attack chain:**
1. Compromise any pod in the cluster (or reach repo-server via network)
2. Send unauthenticated gRPC `GenerateManifest` request with attacker-controlled Git repo
3. Execute arbitrary script on repo-server (via kustomize's --helm-command)
4. Steal Redis password from env → poison deployment cache → deploy attacker workload

**Defense:** Enable Kubernetes network policies (`networkPolicy.create=true` in Helm). Verify with `kubectl get networkpolicy -A`. No patch exists as of Jul 10, 2026 — reported to maintainers in Jan 2025, 18 months ago.

**Related:** CVE-2024-31989 (Redis auth bypass) is revived by this attack — Redis cache is unsigned, so stealing the password reopens the same vector.

**Reference:** https://thehackernews.com/ (search "Unpatched Argo CD Repo-Server Flaw")

## 🟡 High / Medium

### 2. CVE-2026-3864 — CSI Driver for NFS Path Traversal
**Source:** K8s Security Announcements (Jul 04, 2026)
**CVSS:** 6.5 (Medium)
**Component:** Kubernetes CSI Driver for NFS — all versions prior to v4
**Status:** ✅ Fix available in v4+

A path traversal vulnerability exists in the CSI Driver for NFS where insufficient validation of the `subDir` parameter in volume identifiers allows a malicious user to craft a `volumeHandle` containing traversal sequences (`../`). During volume deletion cleanup operations, the driver may operate on unintended directories on the NFS server, potentially causing data loss.

**Impact:** A user with PersistentVolume creation privileges referencing the NFS CSI driver can delete directories outside the intended volume path on the NFS server.

**Mitigation:** Upgrade to CSI Driver for NFS v4 or later.

### 3. CVE-2026-34040 — Docker Engine AuthZ Bypass (Container Escape)
**Source:** The Hacker News (Mar 24, 2026) — relevant for K8s container runtimes
**CVSS:** 8.8 (High)
**Component:** Docker Engine authorization plugins (AuthZ)

An incomplete fix for CVE-2024-41110 allows attackers to bypass authorization plugins under specific circumstances. Relevant to Kubernetes clusters using Docker as container runtime. Docker Engine users should ensure they're on the latest patched version.

### 4. CVE-2026-43503 — DirtyClone Linux Kernel LPE (Container Breakout Risk)
**Source:** The Hacker News (Jun 26, 2026)
**CVSS:** 8.8 (High)
**Component:** Linux kernel — network packet cloning path (DirtyFrag family)

A local privilege escalation in the Linux kernel that lets a local user corrupt file-backed memory through cloned network packets. Relevant to Kubernetes nodes — a container escape followed by DirtyClone gives full root on the host. Patch landed in mainline May 21, 2026. CVE-2026-43503.

**Action:** Update Linux kernel to include the May 21 patch.

### 5. CVE-2020-8561 — API Server Webhook Redirect (Reclassified as Unfixed)
**Source:** Kubernetes Blog (May 26, 2026)
**CVSS:** 4.1 (Medium)
**Component:** kube-apiserver admission webhooks

**New development:** The K8s Security Response Committee has corrected CVE records to reflect that this vulnerability (and two others) are **unfixable architectural trade-offs**, not fixed bugs. Starting June 1, 2026, vulnerability scanners will flag these.

The kube-apiserver follows HTTP redirects from admission webhooks, allowing an actor who can configure `AdmissionWebhookConfiguration` to redirect API server requests to internal/private networks.

**Mitigation:** Set API server log level < 10 and disable dynamic profiling (`--profiling=false`).

## ℹ️ Low / Notable / Info

### 6. CVE-2020-8562 — API Server Proxy DNS TOCTOU (Reclassified as Unfixed)
**Source:** Kubernetes Blog (May 26, 2026)
**CVSS:** 3.1 (Low)
**Component:** kube-apiserver proxy

A TOCTOU race condition in the API server proxy allows IP restriction bypass. Reclassified as unfixed on June 1, 2026 — will now appear in vulnerability scanners.

**Mitigation:** Use local DNS caching (dnsmasq) for the API server with `min-cache-ttl` configured.

### 7. CVE-2021-25740 — Cross-Namespace Forwarding via Endpoints (Reclassified as Unfixed)
**Source:** Kubernetes Blog (May 26, 2026)
**CVSS:** 3.1 (Low)
**Component:** Endpoints / EndpointSlice API

Users can manually specify IP addresses in Endpoint objects to point LoadBalancers or Ingresses toward backends in other namespaces. Reclassified as unfixed on June 1, 2026.

**Mitigation:** Restrict write access to Endpoints and EndpointSlices via RBAC. Since K8s v1.22, default `edit` and `admin` ClusterRoles no longer include these permissions.

### 8. "Reconciling the Past" — CVE Record Corrections
**Source:** Kubernetes Blog (May 26, 2026)
**Authors:** Pushkar Joglekar (Broadcom/SIG Security), Tabitha Sable (Datadog/K8s SRC/SIG Security)

The Kubernetes Security Response Committee corrected CVE records for CVE-2020-8561, CVE-2020-8562, CVE-2021-25740, and CVE-2020-8554 to accurately reflect they are **unfixed** (architectural design trade-offs, not patchable bugs). Effective June 1, 2026.

**Why this matters:** Vulnerability scanners will now detect these in all versions where they previously reported "not affected." Administrators should ensure mitigations are in place before scanners start reporting these.

- **TeamPCP** continues targeting cloud-native tooling: Trivy, KICS, LiteLLM compromised via CI/CD pipeline attacks (Feb–Mar 2026). GitHub itself was breached in May 2026 via employee device compromise (3,800+ internal repos exfiltrated).

- The K8s project now explicitly documents **4 unfixed CVEs** as architectural design decisions. Expect more as OSV file generation matures.
- **Action:** Review CVE-2020-8561, CVE-2020-8562, CVE-2021-25740, CVE-2020-8554 mitigations. Run vulnerability scanner on clusters after June 1 to verify expected detections.

- **Action:** If you run Argo CD, verify network policies are active immediately. The Helm chart ships with them off by default.
