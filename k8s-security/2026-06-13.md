## 🔴 Latest CVEs

### 1. 🟡 CVE-2026-3865 — CSI Driver for SMB path traversal via subDir
- **Published:** Apr 11, 2026
- **Severity:** Medium (CVSS 6.5) — `AV:N/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:H`
- **Description:** Insufficient validation of the `subDir` parameter in volume identifiers allows path traversal. A user with PersistentVolume creation privileges can craft a `volumeHandle` with `../` sequences, causing the driver to delete/modify directories outside the intended subdirectory on the SMB server.
- **Affected:** All versions < v1.20.1
- **Fix:** Upgrade to CSI Driver for SMB v1.20.1+
- **Reporter:** Shaul Ben Hai (SentinelOne)
- **Detection:** Inspect `volumeHandle` fields for `../` sequences; review CSI controller logs for unexpected directory operations.
- **GitHub:** https://github.com/kubernetes/kubernetes/issues/138319

### 2. 🟡 CVE-2026-4342 — ingress-nginx comment-based nginx configuration injection
- **Published:** Mar 20, 2026
- **Severity:** HIGH (CVSS 8.8) — `AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H`
- **Description:** A combination of Ingress annotations can be used to inject configuration into nginx, leading to arbitrary code execution in the ingress-nginx controller and disclosure of Secrets accessible to the controller (which in default installations is all Secrets cluster-wide).
- **Affected:**
  - ingress-nginx: < v1.13.9
  - ingress-nginx: < v1.14.5
  - ingress-nginx: < v1.15.1
- **Fix:** Upgrade to v1.13.9, v1.14.5, or v1.15.1+
- **Reporter:** wooseokdotkim
- **Detection:** Suspicious data in the `rules.http.paths.path` field of an Ingress resource.
- **GitHub:** https://github.com/kubernetes/kubernetes/issues/137893

## 🔮 Trends & Recommendations

1. **TeamPCP is the dominant threat** — This group is systematically compromising open-source tools (Trivy, LiteLLM, Checkmarx actions) via CI/CD credential theft, then using those footholds to deploy K8s lateral movement toolkits. **Mitigation:** Audit CI/CD token permissions, enable OIDC-based auth where possible, monitor for unexpected GitHub Action runs.

2. **ingress-nginx remains the most actively vulnerable K8s component** — 2026 has seen 6+ CVEs so far (CVE-2026-4342, CVE-2026-3288, CVE-2026-1580, CVE-2026-24512-14). **Mitigation:** Stay current on ingress-nginx versions. Current recommended: v1.15.1+.

3. **Worm-like attacks targeting exposed K8s APIs are increasing** — PCPJack and the TeamPCP worm specifically target exposed Docker APIs and K8s clusters. **Mitigation:** Never expose K8s API server to the public internet, use network policies, enable audit logging.

4. **Architectural debt CVEs going "unfixed"** — The K8s project is now formally documenting design trade-offs as permanent risks rather than issuing incomplete patches. **Action:** Review CVE-2020-8561, CVE-2020-8562, and CVE-2021-25740 mitigations.

5. **CSI storage drivers need attention** — Both SMB (CVE-2026-3865) and NFS (CVE-2026-3864) CSI drivers had path traversal bugs. **Mitigation:** Upgrade CSI drivers and restrict PV creation privileges.
