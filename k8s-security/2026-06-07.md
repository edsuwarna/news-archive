## 1. 🔴🔴 Latest CVEs

### 🟡 CVE-2026-4342 — ingress-nginx Comment-Based Nginx Configuration Injection
**Severity: HIGH (CVSS 8.8) — RCE + Secret Disclosure**
A combination of Ingress annotations can be used to inject configuration into nginx. Leads to arbitrary code execution in the context of the ingress-nginx controller and disclosure of Secrets accessible cluster-wide.
- **Affected:** ingress-nginx v1.13.9, v1.14.5, v1.15.1
- **Fixed:** Same versions listed (patch release)
- **Mitigation:** Upgrade ingress-nginx to latest patch
- **Detection:** Suspicious data in `rules.http.paths.path` field of Ingress resources

### 🟡 CVE-2026-3288 — ingress-nginx rewrite-target Nginx Configuration Injection
**Severity: HIGH (CVSS 8.8) — RCE + Secret Disclosure**
The `nginx.ingress.kubernetes.io/rewrite-target` annotation can be used to inject nginx configuration. Leads to arbitrary code execution and Secret disclosure.
- **Affected:** ingress-nginx v1.13.8, v1.14.4, v1.15.0
- **Fixed:** Same versions listed
- **Mitigation:** Use admission control to block `rewrite-target` annotation, then upgrade

### 🟡 CVE-2026-1580, CVE-2026-24512, CVE-2026-24513, CVE-2026-24514 — Multiple ingress-nginx Issues
**Severity: HIGH (CVSS 8.8)** — Multiple vulnerabilities disclosed in ingress-nginx, the most serious rated HIGH (8.8).
- **Affected:** ingress-nginx v1.13.7, v1.14.3
- **Fixed:** v1.13.7+, v1.14.3+

### 🟡 CVE-2025-7342 — K8s Image Builder Default Credentials (Nutanix/OVA)
**Severity: HIGH (CVSS 8.1) — Unauthorized SSH/RDP Access**
Windows VM images built with Kubernetes Image Builder v0.1.44 or earlier using Nutanix or OVA providers ship with default credentials. An unauthorized user can SSH/RDP/WINRM into nodes.
- **Affected:** Image Builder ≤ v0.1.44 (Nutanix/OVA Windows images)
- **Mitigation:** Rebuild VM images with patched Image Builder

### 🟡 CVE-2026-3865 — CSI Driver for SMB Path Traversal via subDir
**Severity: Medium (CVSS 6.5) — Arbitrary Directory Deletion on SMB Server**
Insufficient validation of `subDir` parameter in volume identifiers allows path traversal (`../` sequences) during volume deletion. Attacker with PersistentVolume creation privileges can delete/modify directories outside the managed subdirectory.
- **Affected:** All SMB CSI driver versions prior to v1.20.1
- **Fixed:** v1.20.1
- **Mitigation:** Upgrade driver, restrict PV creation to trusted admins

### 🟡 CVE-2026-3864 — CSI Driver for NFS Path Traversal via subDir
**Severity: Medium (CVSS 6.5) — Arbitrary Directory Deletion on NFS Server**
Same pattern as CVE-2026-3865 but for the NFS CSI driver. Volume deletion cleanup follows traversal sequences in volumeHandle.
- **Affected:** All NFS CSI driver versions prior to v4.13.1
- **Fixed:** v4.13.1
- **Mitigation:** Upgrade driver, restrict PV creation to trusted admins

### ℹ️ CVE-2025-7445 — secrets-store-sync-controller Discloses SA Tokens in Logs
**Severity: Medium (CVSS 6.5)**
Service account tokens leaked in controller logs during certain error conditions. Tokens could be exchanged with external cloud providers to access vault secrets.
- **Affected:** secrets-store-sync-controller v0.0.2
- **Fixed:** v0.0.2+
- **Detection:** `kubectl logs ... -c manager | grep csi.storage.k8s.io/serviceAccount.tokens`

### ℹ️ CVE-2025-5187 — Nodes Can Delete Themselves via OwnerReference
**Severity: Medium (CVSS 6.7)**
Node users can patch themselves with an OwnerReference to a cluster-scoped resource. If the referenced resource doesn't exist or is deleted, garbage collection removes the node. Compromised nodes can delete+recreate their node object with modified taints/labels.
- **Affected:** kube-apiserver v1.31.11, v1.32.7, v1.33.3
- **Fixed:** Later patches in each release line
- **Mitigation:** Enable `OwnerReferencesPermissionEnforcement` admission controller

### ℹ️ CVE-2025-14269 — Credential Caching in Headlamp with Helm Enabled
Credential caching issue in the in-cluster version of Headlamp when Helm is enabled.

### ℹ️ CVE-2025-13281 — Portworx Half-Blind SSRF in kube-controller-manager
A half-blind Server Side Request Forgery (SSRF) vulnerability in kube-controller-manager related to Portworx.

### ℹ️ CVE-2025-9708 — Kubernetes C# Client Improper Certificate Validation
Incorrect certificate validation in custom CA mode may lead to man-in-the-middle attacks.

### ℹ️ CVE-2025-4563 — Nodes Can Bypass Dynamic Resource Allocation Authorization Checks
Nodes can bypass DRA authorization checks.

### ℹ️ CVE-2025-0426 — Node Denial of Service via kubelet Checkpoint API
Large number of checkpoint requests can cause node DoS.

### ℹ️ CVE-2024-9042 — Windows Node Command Injection via /logs API (CVSS 5.9)
User with ability to query node's `/logs` endpoint can execute arbitrary commands on the host.

## 2. 🔴🔴 Security News

### 🔴 K8s Blog — Reconciling the Past: Correcting Records for Unfixed Kubernetes CVEs
*May 26, 2026 | kubernetes.io*

The Kubernetes Security Response Committee (SRC) corrected CVE records on **June 1, 2026** for vulnerabilities that were previously incorrectly marked as having fixes. **These CVEs are now officially documented as unfixed** — this is an architectural decision, not a regression.

**CVEs affected:**
- **CVE-2020-8561 (Medium, 4.1)** — Webhook redirect in kube-apiserver. API server follows HTTP redirects to internal networks. Mitigation: set log level < 10, disable profiling.
- **CVE-2020-8562 (Low, 3.1)** — DNS TOCTOU race in API server proxy bypass. Mitigation: use local DNS cache with `min-cache-ttl`.
- **CVE-2021-25740 (Low, 3.1)** — Cross-namespace forwarding via Endpoints. Mitigation: restrict Endpoints write access (default edit/admin ClusterRoles changed since 1.22).
- **CVE-2020-8554 (no score change)** — Already correctly listed as unfixed; version number format updated.

⚠️ **Action required:** Vulnerability scanners will now flag these CVEs in environments where they were previously ignored. Review your scanning and alerting baselines.

## 3. 🔴 Trends and Recommendations

### 🔴 Ingress-NGINX Under Intense Fire
The ingress-nginx project has seen an unprecedented volume of security disclosures in 2025-2026, with **at least 7 distinct CVEs** covering annotation injection, configuration injection, and code execution. This makes ingress-nginx the single largest attack surface in many clusters. **Recommendation:** Evaluate Gateway API as a replacement. The `ingress2gateway` migration tool reached v1.0. Consider network policies and admission controllers as defense-in-depth while migrating.

### 🔴 TeamPCP — The Dominant Cloud-Native Threat Actor
TeamPCP continues to be the most active threat actor targeting the Kubernetes ecosystem:
- Compromised **Trivy** (Aqua Security's scanner) — malicious Docker Hub images
- Compromised **LiteLLM** — backdoored PyPI packages
- Compromised **Checkmarx GitHub Actions** — stolen CI credentials
- Deployed **PCPJack** credential stealer — worm-like propagation across cloud environments
- Targeted **Docker Hub, Checkmarx, KICS, Litellm**, and now GitHub itself

**Recommendation:** Audit all CI/CD pipelines for exposed credentials. Pin image digests, not tags. Implement software attestation (SLSA, in-toto).

### 🟡 CVE Record Corrections — Prepare for Scanner Noise
The Kubernetes SRC's June 1 correction of CVE records (CVE-2020-8561, CVE-2020-8562, CVE-2021-25740, CVE-2020-8554) to "unfixed" status will cause a wave of new scanner findings in environments that were previously passing. **Recommendation:** Pre-audit these CVEs in your clusters, document accepted risks, and configure scanner exception rules before the alerts arrive.

### 🟡 Docker AuthZ Bypass — Incomplete Fix Pattern
CVE-2026-34040 is an incomplete fix for CVE-2024-41110, highlighting the difficulty of properly hardening Docker's authorization plugin mechanism. **Recommendation:** Monitor for additional bypass variants. Consider replacing Docker's AuthZ with eBPF-based runtime security (Falco, Tetragon) for container authorization.

### ℹ️ CSI Driver Path Traversal — Pattern Emerging
Two CVEs (CVE-2026-3865 for SMB, CVE-2026-3864 for NFS) share the exact same vulnerability pattern — path traversal via `subDir` parameter during volume deletion operations. This suggests the CSI framework itself may have a design gap. **Recommendation:** Pin CSI driver versions and audit any `subDir` usage in PV configurations.
