## 🔴 Critical

### ⚡ PCPJack Credential Stealer Exploits 5 CVEs to Spread Worm-Like Across Cloud Systems
- **Date:** May 07, 2026
- **Source:** The Hacker News / SentinelOne
- **Summary:** A new credential theft framework called **PCPJack** is actively targeting exposed cloud infrastructure (Docker, Kubernetes, Redis, MongoDB, RayML) and aggressively removing artifacts from a rival threat group (TeamPCP). It exploits **5 CVEs** to propagate in a worm-like fashion and harvests credentials from cloud, container, developer, productivity, and financial services.
- **CVEs exploited:** CVE-2025-55182, CVE-2025-29927, CVE-2026-1357, CVE-2025-9501, CVE-2025-48703
- **Key payloads:** `worm.py` (orchestrator + Telegram C2), `lateral.py` (K8s/Docker/SSH lateral movement), `cloud_scan.py` (port scanning from Common Crawl data), `parser.py` (credential extraction), `crypto_util.py` (exfiltration encryption)
- **Action:** Harden exposed API servers, audit K8s RBAC, restrict network access to control plane components, review exposed Docker/Redis/MongoDB instances.

## 🟡 High / Medium

### CVE-2026-3865 — CSI Driver for SMB path traversal via subDir (CVSS 6.5 Medium)
- **Date:** April 11, 2026
- **Component:** `smb.csi.k8s.io` (SMB CSI Driver)
- **Summary:** Insufficient validation of the `subDir` parameter in volume identifiers allows path traversal. A user with `PersistentVolume` create permissions can craft a `volumeHandle` containing `../` traversal sequences. During volume deletion cleanup, the driver may delete/modify directories outside the intended managed subdirectory on the SMB server.
- **Affected:** Clusters running the SMB CSI driver where users can create PVs referencing it.
- **CVSS:** 6.5 — AV:N/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:H
- **Action:** Upgrade the CSI Driver for SMB to the latest version.

### CVE-2026-3864 — CSI Driver for NFS path traversal via subDir (CVSS 6.5 Medium)
- **Date:** March 17, 2026
- **Component:** `nfs.csi.k8s.io` (NFS CSI Driver)
- **Summary:** Identical vulnerability pattern to CVE-2026-3865 but affecting the NFS CSI driver. Path traversal via `subDir` parameter allows unintended directory deletion/modification on the NFS server during volume cleanup.
- **Action:** Upgrade the CSI Driver for NFS to the latest version.

### CVE-2026-4342 — ingress-nginx comment-based nginx configuration injection (CVSS 8.8 High)
- **Date:** March 20, 2026
- **Component:** ingress-nginx
- **Summary:** A combination of Ingress annotations can be used to inject configuration into nginx, leading to arbitrary code execution in the context of the ingress-nginx controller and disclosure of Secrets accessible cluster-wide.
- **Affected versions:** ingress-nginx < v1.13.9, < v1.14.5, < v1.15.1
- **CVSS:** 8.8 — AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **Action:** **Upgrade immediately** to ingress-nginx v1.13.9+, v1.14.5+, or v1.15.1+.

### CVE-2026-3288 — ingress-nginx rewrite-target nginx configuration injection (CVSS 8.8 High)
- **Date:** March 10, 2026
- **Component:** ingress-nginx
- **Summary:** The `nginx.ingress.kubernetes.io/rewrite-target` annotation can be used to inject configuration into nginx, leading to arbitrary code execution and Secret disclosure (cluster-wide in default installations).

## ℹ️ Notable / Info

### Reconciling the Past: Correcting Records for Unfixed Kubernetes CVEs
- **Date:** May 26, 2026 (updated June 1, 2026)
- **Source:** Kubernetes Blog
- **Summary:** The Kubernetes SRC has corrected CVE records for **four unfixed vulnerabilities** that had incorrectly listed fixed versions. These are architectural design trade-offs that cannot be fully remediated through code:
  - **CVE-2020-8561** (Medium 4.1): Webhook redirect in kube-apiserver — restrict log verbosity and disable profiling.
  - **CVE-2020-8562** (Low 3.1): Proxy bypass via DNS TOCTOU race condition — use local DNS caching (dnsmasq).
  - **CVE-2021-25740** (Low 3.1): Cross-namespace forwarding via Endpoints — harden RBAC for Endpoints/EndpointSlices.
  - **CVE-2020-8554** (previously documented): Service IP spoofing — record updated to standardized format.
- **Impact:** Vulnerability scanners may now flag these in places they weren't detected before. Administrators should implement the recommended mitigations.

## Trends & Recommendations

1. **ingress-nginx remains the most active CVE target in the K8s ecosystem** — 5+ distinct CVEs in 2026 alone. The project's retirement (announced March 2026) means no new feature development, but urgent patches continue. **Migrate to Gateway API + a supported ingress controller** (e.g., Contour, Istio, NGINX Ingress Controller maintained by NGINX Inc., or Envoy Gateway) as soon as feasible.

2. **CSI driver supply chain scrutiny increasing** — Both SMB and NFS CSI drivers had near-identical path traversal vulnerabilities in consecutive months. Audit any CSI driver that accepts user-supplied path parameters.

3. **Cloud credential theft frameworks are maturing** — PCPJack represents a new generation of modular malware specifically designed for multi-cloud environments, using Common Crawl data for target discovery and Telegram for C2. Defenders should monitor for exposed Docker/K8s API servers and enforce network segmentation.

4. **CVE record hygiene improving** — The K8s SRC's correction of unfixed CVE records signals a commitment to accurate vulnerability data. Expect more scanners to flag these architectural issues; plan mitigations proactively.

5. **Helm chart security** — Microsoft's guidance on default Helm chart configurations remains critical. Implement policy-as-code (e.g., OPA/Gatekeeper) to prevent deployment of misconfigured charts.
