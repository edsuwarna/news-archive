# Kubernetes Security News — June 22, 2026

## Latest CVEs

### 🟡 CVE-2026-3865 — CSI Driver for SMB Path Traversal via subDir (Apr 11)
- **Severity:** Medium (6.5) — CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:H
- **Description:** Insufficient validation of the `subDir` parameter in volume identifiers enables path traversal. A malicious user with PersistentVolume creation privileges can craft a `volumeHandle` containing `../` sequences to delete/modify directories outside the intended subdirectory on the SMB server.
- **Affected:** All versions < v1.20.1
- **Fixed:** CSI Driver for SMB >= v1.20.1
- **Reported by:** @Shaul Ben Hai (SentinelOne)
- **Detection:** Inspect PersistentVolumes for traversal sequences in `volumeHandle`; review CSI controller logs for unexpected directory operations.

### 🟡 CVE-2026-3864 — CSI Driver for NFS Path Traversal via subDir (Mar 17)
- **Severity:** Medium (6.5) — CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:H
- **Description:** Identical vulnerability pattern as CVE-2026-3865 but in the NFS CSI driver. Insufficient subDir validation allows path traversal during volume deletion.
- **Affected:** All versions < v4.13.1
- **Fixed:** CSI Driver for NFS >= v4.13.1
- **Reported by:** @Shaul Ben Hai (SentinelOne)
- **Detection:** Same as CVE-2026-3865 — inspect volumeHandle fields and controller logs.

### 🟡 CVE-2026-4342 — ingress-nginx Comment-Based nginx Configuration Injection (Mar 20)
- **Severity:** HIGH (8.8) — CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **Description:** A combination of Ingress annotations can be used to inject configuration into nginx, leading to arbitrary code execution in the context of the ingress-nginx controller and disclosure of Secrets accessible to the controller.
- **Affected:** ingress-nginx < v1.13.9, < v1.14.5, < v1.15.1
- **Fixed:** ingress-nginx v1.13.9, v1.14.5, v1.15.1
- **Reported by:** wooseokdotkim
- **Detection:** Suspicious data within `rules.http.paths.path` field of Ingress resources.

### 🟡 CVE-2026-3288 — ingress-nginx rewrite-target nginx Configuration Injection (Mar 10)
- **Severity:** HIGH (8.8) — CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **Description:** The `nginx.ingress.kubernetes.io/rewrite-target` annotation can be used to inject configuration into nginx, enabling arbitrary code execution and Secret disclosure.
- **Affected:** ingress-nginx < 1.13.8, < 1.14.4, < 1.15.0
- **Fixed:** ingress-nginx 1.13.8, 1.14.4, 1.15.0
- **Reported by:** Kai Aizen
- **Mitigation:** Block the use of rewrite-target annotation via admission control prior to upgrading.

### 🟡 Multiple ingress-nginx CVEs (Feb 2, 2026)
- **CVEs:** CVE-2026-1580, CVE-2026-24512, CVE-2026-24513, CVE-2026-24514
- **Severity:** HIGH (8.8) for the most serious
- **Description:** Four issues disclosed together in ingress-nginx.
- **Affected:** ingress-nginx < v1.13.7, < v1.14.3
- **Fixed:** ingress-nginx v1.13.7, v1.14.3

### ℹ️ Unfixed CVEs — CVE Records Corrected (June 1, 2026)
- **CVE-2020-8561** — Webhook redirect in kube-apiserver. Medium (4.1). Unfixable architectural limitation. Mitigation: restrict log verbosity (`--v < 10`), disable profiling.
- **CVE-2020-8562** — Proxy bypass via DNS TOCTOU. Low (3.1). Unfixable. Mitigation: deploy dnsmasq caching resolver on control plane nodes.
- **CVE-2021-25740** — Cross-namespace forwarding via Endpoints. Low (3.1). Unfixable. Mitigation: hardened RBAC to remove Endpoints write access from broad roles.
- **CVE-2020-8554** — Services using external IPs. Unfixable. Affects all versions.
- **Note:** The K8s SRC updated these CVE records on June 1, 2026 to correctly reflect "unfixed" status. Vulnerability scanners will now flag these where they were previously undetected.

## Security News

### 🔴 GitHub Breached — TeamPCP Exfiltrates 3,800+ Internal Repos (May 20, 2026)
GitHub disclosed unauthorized access to its internal repositories after TeamPCP listed the platform's source code for sale ($50,000+). The breach stemmed from a compromised employee device. TeamPCP claimed ~4,000 repositories were stolen. No evidence of customer data impact so far.

### 🔴 PCPJack Credential Stealer — Worm-Like Cloud Propagation (May 7, 2026)
SentinelOne disclosed PCPJack, a credential theft framework specifically targeting Docker, Kubernetes, Redis, MongoDB, RayML, and vulnerable web applications. Exploits 5 CVEs to spread in a worm-like fashion across cloud systems, harvesting credentials and exfiltrating via attacker-controlled infrastructure.

### 🟡 Docker CVE-2026-34040 — AuthZ Bypass (Apr 7, 2026)
A high-severity vulnerability (CVSS 8.8) in Docker Engine. Incomplete fix for CVE-2024-41110 — attackers can bypass authorization plugins by crafting API requests that cause the Docker daemon to forward requests without the body to authorization plugins.

### 🟡 Trivy Supply Chain Attack — TeamPCP Poisons CI/CD (Mar 23–24, 2026)
TeamPCP compromised the Trivy vulnerability scanner's CI/CD, pushing malicious Docker images (v0.69.4–0.69.6) containing infostealers. The breach spread to:
- **LiteLLM** (versions 1.82.7–1.82.8): three-stage payload — credential harvester, K8s lateral movement toolkit (deploys privileged pods to every node), and persistent systemd backdoor
- **Checkmarx GitHub Actions** (ast-github-action, kics-github-action): stolen CI credentials used to poison additional workflows
- CVE-2026-33634 (CVSS 9.4) tracks the Trivy compromise

### 🔴 TeamPCP Worm Campaign — Cloud Infrastructure Exploitation (Feb 9, 2026)
Massive worm-driven campaign targeting exposed Docker APIs, Kubernetes clusters, Ray dashboards, and Redis servers. Leveraged React2Shell (CVE-2025-55182, CVSS 10.0) to build criminal infrastructure. TeamPCP has over 700 Telegram members and is actively selling stolen data.

### 🟡 Fluent Bit 5-Flaw Chain — RCE in Cloud Infrastructures (Nov 24, 2025)
Oligo Security disclosed 5 vulnerabilities in Fluent Bit (CVE-2025-12972 etc.) enabling auth bypass, path traversal, RCE, DoS, and tag manipulation. Chained exploitation could lead to full cloud infrastructure compromise.

### 🟡 Chaos Mesh — Critical GraphQL Flaws Enable K8s Cluster Takeover (Sep 16, 2025)
JFrog disclosed "Chaotic Deputy" — multiple vulnerabilities in Chaos Mesh (CVE-2025-59358, CVSS 7.5) allowing unauthenticated GraphQL access, RCE, and full cluster takeover with minimal in-cluster network access.

### 🟡 NVIDIA Container Toolkit — CVE-2025-23266, CVSS 9.0 (Jul 18, 2025)
"NVIDIAScape" — critical container escape vulnerability affecting AI cloud services. Affected all NVIDIA Container Toolkit versions up to 1.17.7 and GPU Operator up to 25.3.0. Fixed in 1.17.8 / 25.3.1.

### ℹ️ Microsoft Warns — Default Helm Charts Risk Data Leaks (May 6, 2025)
Default Helm chart configurations prioritize ease-of-use over security, leading to widespread misconfigurations exposing sensitive data and cloud resources.

### 🔴 Ingress NGINX Retired (March 2026)
The Kubernetes Steering Committee and SRC strongly reiterated that Ingress NGINX was retired as of March 2026. **No more security patches or releases will be issued.** ~50% of cloud native environments are affected. Migration to Gateway API or alternative Ingress controllers is critical.

## Trends

### 1. TeamPCP — Dominant Cloud-Native Threat Actor
TeamPCP (aka DeadCatx3, PCPcat) has become the most significant threat to cloud-native infrastructure in 2026. Their operations span:
- **Supply chain attacks:** Trivy → LiteLLM → Checkmarx → GitHub (cascading)
- **Worm-like propagation:** PCPJack + exposed Docker/K8s APIs
- **Credential theft:** SSH keys, cloud credentials, K8s secrets, cryptocurrency wallets
- **K8s lateral movement:** Deploying privileged pods across nodes
- **Monetization:** Sale of stolen data, credential resale, cryptomining

**Recommendation:** Harden exposed Docker APIs and K8s clusters immediately. Restrict egress from cluster nodes. Audit CI/CD pipeline credentials. Monitor for `sysmon.service` backdoor.

### 2. ingress-nginx Attack Surface — Critical After Retirement
With 5 CVEs in 2026 (CVE-2026-4342, CVE-2026-3288, CVE-2026-1580, CVE-2026-24512, CVE-2026-24513, CVE-2026-24514) and the project now retired, organizations still running ingress-nginx face **unpatched zero-day risk**. All 2026 CVEs had severity HIGH (8.8) and involved arbitrary code execution.

**Recommendation:** Migrate to Gateway API or alternative ingress controllers (nginxinc/kubernetes-ingress, Contour, Traefik, Istio) immediately. Audit all existing Ingress resources for suspicious annotations.

### 3. CSI Driver Supply Chain — Path Traversal Pattern
Two CVEs (CVE-2026-3865 SMB, CVE-2026-3864 NFS) show a recurring pattern: insufficient validation in CSI drivers enabling path traversal on storage backends. Both reported by the same researcher (Shaul Ben Hai, SentinelOne).

**Recommendation:** Upgrade CSI drivers promptly. Restrict PersistentVolume creation privileges to trusted administrators.

### 4. CVE Record Transparency — Unfixed Vulnerabilities Formalized
The K8s SRC's correction of CVE records (CVE-2020-8561, CVE-2020-8562, CVE-2021-25740, CVE-2020-8554) marks a shift toward documenting architectural limitations honestly rather than maintaining the illusion of "patch-only" security. Administrators should expect these in vulnerability scanner results starting June 2026.

**Recommendation:** Review and apply recommended mitigations for unfixed CVEs. These are not new vulnerabilities but newly recognized risks that require administrative controls rather than code patches.
