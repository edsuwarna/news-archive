## 🔴 Critical

### 1. Unpatched Argo CD Repo-Server Flaw — Full Cluster Takeover
**Source:** The Hacker News | **Date:** Jul 01, 2026

A critical unauthenticated code execution vulnerability has been disclosed in Argo CD's repo-server component. The internal gRPC service has no authentication — anyone who can reach it can send a crafted request to run arbitrary commands via the `GenerateManifest` service.

**Impact:** An attacker can:
- Execute code on the repo-server (reads Git repos, builds Kubernetes manifests)
- Read the cluster's Redis password from environment variables
- Poison Argo CD's Redis cache to deploy attacker-controlled workloads on the next automatic sync

**Status:** ❗ **No patch available.** Synacktiv (who discovered the flaw and reported it to maintainers ~18 months ago) published details after no fix was released.

**Mitigation:** Apply Kubernetes Network Policies to isolate the repo-server and Redis ports so only Argo CD's own components can reach them. Helm users must enable policies explicitly — the chart leaves them off by default.

**CVE:** None assigned yet (no patch → no CVE)

## 🟡 High / Medium

### 2. CVE-2026-4342 — ingress-nginx Comment-Based Configuration Injection
**Source:** K8s Security Announce | **Date:** Mar 20, 2026 | **CVSS: 8.8 (HIGH)**

A combination of Ingress annotations can be used to inject configuration into nginx, leading to arbitrary code execution in the context of the ingress-nginx controller and disclosure of Secrets accessible to the controller (cluster-wide by default in standard installations).

**Affected:** ingress-nginx (all versions without the fix)

### 3. CVE-2026-3288 — ingress-nginx Rewrite-Target Injection
**Source:** K8s Security Announce | **Date:** Mar 10, 2026 | **CVSS: 8.8 (HIGH)**

The `nginx.ingress.kubernetes.io/rewrite-target` annotation can inject nginx configuration, leading to code execution and Secret disclosure. Same impact profile as CVE-2026-4342.

**Affected:** ingress-nginx (all versions without the fix)

### 5. CVE-2026-3865 — CSI Driver for SMB Path Traversal via subDir
**Source:** K8s Security Announce | **Date:** Apr 11, 2026 | **CVSS: 6.5 (Medium)**

Insufficient validation of the `subDir` parameter in volume identifiers allows path traversal. An attacker with PV creation privileges can craft a `volumeHandle` containing `../` sequences to delete or modify directories outside the intended managed subdirectory on the SMB server.

### 6. CVE-2026-3864 — CSI Driver for NFS Path Traversal via subDir
**Source:** K8s Security Announce | **Date:** Mar 17, 2026 | **CVSS: 6.5 (Medium)**

Same vulnerability pattern as CVE-2026-3865 but for the NFS CSI driver.
