## 🔴 Critical

### Unpatched Argo CD Repo-Server Flaw — Full Cluster Takeover
- **Date:** 2026-07-01
- **Source:** The Hacker News
- **Details:** Synacktiv disclosed an unpatched flaw in Argo CD's repo-server component that lets an **unauthenticated attacker execute code** if they can reach the component's internal network port. Leads to **full Kubernetes cluster takeover**. No fix available and no CVE assigned yet. Argo CD maintainers notified.
- **Impact:** Any cluster with Argo CD (repo-server port exposed internally) is vulnerable.
- **Action:** Monitor Argo CD security channels; restrict network access to repo-server to only Argo CD components.

## 🟡 High / Medium

### CVE-2026-4342 — ingress-nginx Comment-Based Nginx Configuration Injection
- **CVSS:** 8.8 (HIGH) — CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **Date:** 2026-03-20
- **Source:** K8s Security Announce
- **Details:** A combination of Ingress annotations can inject configuration into nginx. Leads to **arbitrary code execution** in the ingress-nginx controller and **disclosure of all Secrets** accessible to the controller (default: all Secrets cluster-wide).
- **Action:** Upgrade ingress-nginx immediately.

### CVE-2026-3288 — ingress-nginx Rewrite-Target Injection (RCE)
- **CVSS:** 8.8 (HIGH) — CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **Date:** 2026-03-10
- **Source:** K8s Security Announce
- **Details:** The `nginx.ingress.kubernetes.io/rewrite-target` annotation can inject configuration into nginx. Same impact class as CVE-2026-4342: RCE + Secrets disclosure.
- **Action:** Upgrade ingress-nginx to the latest version.

### Multiple ingress-nginx Issues (4 CVEs) — HIGH
- **CVSS:** 8.8 (HIGH)
- **Date:** 2026-02-02
- **Source:** K8s Security Announce
- **Details:** Four CVEs disclosed simultaneously: **CVE-2026-1580**, **CVE-2026-24512**, **CVE-2026-24513**, **CVE-2026-24514**. All rated HIGH (8.8), affecting:
  - ingress-nginx < v1.13.7
  - ingress-nginx < v1.14.3

### CVE-2026-34040 — Docker Engine AuthZ Bypass
- **CVSS:** 8.8 (HIGH)
- **Date:** 2026-03-24
- **Source:** The Hacker News
- **Details:** An **incomplete fix for CVE-2024-41110** (max-severity auth bypass). Lets attackers bypass **Authorization plugins (AuthZ)** under specific circumstances. Affects Docker Engine.
- **Impact:** Compromised containers can bypass authorization controls.
- **Action:** Update Docker Engine to the latest patched version.

### CVE-2026-43503 — "DirtyClone" Linux Kernel Privilege Escalation
- **CVSS:** 8.8 (HIGH)
- **Date:** 2026-06-26
- **Source:** The Hacker News / JFrog
- **Details:** A new Linux kernel privilege escalation in the DirtyFrag family. Lets a **local user corrupt file-backed memory through a cloned network packet** to gain root. Working exploit published by JFrog.
- **K8s Impact:** Affects K8s node security — a user with pod access (or container breakout) can escalate to node root.
- **Action:** Patch node kernels; update Kubernetes node images.

### CVE-2026-3865 — CSI Driver for SMB Path Traversal
- **CVSS:** 6.5 (Medium) — CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:H
- **Date:** 2026-04-11
- **Source:** K8s Security Announce
- **Details:** Insufficient validation of the `subDir` parameter in SMB CSI volume identifiers allows **path traversal**. A user with PersistentVolume creation privileges can craft `volumeHandle` with `../` sequences, causing the driver to **delete or modify directories outside the intended SMB export subdirectory**.
- **Action:** Update the SMB CSI driver; audit PersistentVolumes for suspicious `volumeHandle` values.

### CVE-2026-3864 — CSI Driver for NFS Path Traversal
- **CVSS:** 6.5 (Medium) — CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:N/I:H/A:H
- **Date:** 2026-03-17
- **Source:** K8s Security Announce
- **Details:** Same pattern as CVE-2026-3865 but for the **NFS CSI driver**. Path traversal via `subDir` allows directory deletion/modification on the NFS server.
- **Action:** Update the NFS CSI driver.

### CVE-2025-14269 — Headlamp Credential Caching (with Helm enabled)
- **CVSS:** N/A (Medium)

- **Details:** In-cluster version of **Headlamp** (Kubernetes Dashboard successor) with Helm enabled caches credentials insecurely. Ties to the Dashboard-to-Headlamp transition blog post (Jun 2026).

## ℹ️ Notable / Info

### Kubernetes CVE Record Reconciliation — Unfixed CVEs Now in Scanner Results
- **Date:** 2026-05-26 (Records updated June 1, 2026)
- **Source:** Kubernetes Blog
- **Details:** The Kubernetes Security Response Committee (SRC) corrected CVE records for **four unfixed architectural issues** that previously had incorrect "fixed version" fields:
  - **CVE-2020-8561** (Medium 4.1) — Webhook redirect in kube-apiserver. Mitigation: limit log verbosity, disable profiling.
  - **CVE-2020-8562** (Low 3.1) — Proxy bypass via DNS TOCTOU. Mitigation: use local dnsmasq caching resolver.
  - **CVE-2021-25740** (Low 3.1) — Cross-namespace forwarding via Endpoints. Mitigation: restrict Endpoints write access.
  - **CVE-2020-8554** — Unfixed, record format standardized.
- **Impact:** Vulnerability scanners will now flag these issues where previously they didn't. Administrators should review their mitigation status.

## Trends & Recommendations

### 1. ingress-nginx CVE Wave Continues (Late 2025–2026)
The retirement of ingress-nginx (announced Jan 2026, EOL March 2026) has been accompanied by a sustained wave of high-severity CVEs. Five CVEs rated HIGH (8.8) were published between Feb–Mar 2026 alone. **Migration to Gateway API / ingress2gateway is strongly recommended** for organizations still on ingress-nginx.

### 2. Supply Chain Attacks Targeting K8s Tooling
The TeamPCP campaign represents the most significant ongoing supply chain threat to the Kubernetes ecosystem — targeting CI/CD pipelines (Trivy, Checkmarx), ML tooling (LiteLLM), and using **Kubernetes-native lateral movement**. This is a shift from generic cryptomining to targeted credential theft.

### 3. K8s 1.36 Security Enhancements
Recent K8s releases brought important security features to GA:
- **User Namespaces** (GA) — better pod isolation
- **Fine-Grained Kubelet API Authorization** (GA)
- **Admission Policies that Can't Be Deleted** — protection against cluster-admin privilege abuse

### 4. CVE Record Transparency Maturation
The K8s SRC's correction of unfixed CVE records (May 2026) signals a shift toward more transparent vulnerability documentation. Administrators should expect to see previously-hidden unfixed issues surface in scanner results.
