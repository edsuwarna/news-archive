## 🔴 Critical / High Severity CVEs (2026)

### 🟡 CVE-2026-3865 — CSI Driver for SMB Path Traversal via subDir (Apr 11)
- **Component:** CSI Driver for SMB
- **Description:** Insufficient validation of the `subDir` parameter in volume identifiers allows path traversal. An attacker with PersistentVolume create access can craft a `volumeHandle` containing `../` sequences. During volume deletion cleanup, the driver may operate on unintended directories on the SMB server.
- **Severity:** Medium
- **Mitigation:** Upgrade CSI Driver for SMB to latest version.

### 🟡 CVE-2026-4342 — ingress-nginx Comment-Based Config Injection (Mar 20)
- **Component:** ingress-nginx
- **CVSS:** 8.8 (HIGH) — CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **Description:** A combination of Ingress annotations can inject configuration into nginx, leading to arbitrary code execution in the ingress-nginx controller context and disclosure of Secrets accessible to the controller (cluster-wide in default installations).
- **Affected:** ingress-nginx < v1.13.9, < v1.14.5, < v1.15.1
- **Fixed:** v1.13.9, v1.14.5, v1.15.1
- **Detection:** Suspicious data in `rules.http.paths.path` field of Ingress resources.

### 🟡 CVE-2026-3864 — CSI Driver for NFS Path Traversal via subDir (Mar 17)
- **Component:** CSI Driver for NFS
- **Description:** Similar to CVE-2026-3865 — path traversal via `subDir` in volume identifiers during cleanup operations on NFS servers.
- **Severity:** Medium
- **Mitigation:** Upgrade CSI Driver for NFS to latest version.

### 🟡 CVE-2026-3288 — ingress-nginx rewrite-target Config Injection (Mar 10)
- **Component:** ingress-nginx
- **CVSS:** 8.8 (HIGH) — CVSS:3.1/AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H
- **Description:** The `nginx.ingress.kubernetes.io/rewrite-target` annotation can inject configuration into nginx, enabling arbitrary code execution and Secret disclosure.
- **Affected:** ingress-nginx < 1.13.8, < 1.14.4, < 1.15.0
- **Fixed:** 1.13.8, 1.14.4, 1.15.0
- **Mitigation (pre-upgrade):** Use admission control to block the rewrite-target annotation.

### 🟡 Multiple ingress-nginx Issues — Feb 2 Batch (Feb 2)

## Trends & Recommendations

1. **🟡 TeamPCP is the dominant threat.** Their modus operandi: compromise OSS tools through CI/CD credential theft → push trojanized packages → deploy credential harvesters → spread worm-like across cloud infrastructure. **Mitigation:** Audit CI/CD secrets rotation, implement SLSA levels, monitor for unexpected Docker Hub pushes from your org, use admission controllers to block privileged pods.

2. **🟡 ingress-nginx remains the most targeted K8s component.** 3 separate CVE batches in 5 months (Feb, Mar 2026) with CVSS 8.8 code injection vulnerabilities. **Mitigation:** Keep ingress-nginx updated (latest v1.15.1+), use admission webhooks to validate annotations, restrict who can create Ingress resources.

3. **ℹ️ CSI driver path traversal** — Both NFS and SMB drivers had similar `subDir` validation gaps. If you use either, ensure you're on the latest version.

4. **🔴 Docker API exposure is the #1 initial access vector.** PCPJack, Dero mining malware, and TeamPCP worm all exploit exposed Docker APIs. **Mitigation:** Never expose Docker API without TLS + AuthZ plugin, use network policies, audit for misconfigured instances.

5. **🟡 Supply chain attacks are accelerating.** Trivy → LiteLLM → Checkmarx demonstrates cascading compromise chains. **Mitigation:** Pin versions with checksums, monitor for unexpected releases, use OSSF Scorecard, consider Chainguard or similar trusted registries.
