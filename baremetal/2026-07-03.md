## 1. 🔥 AMD EPYC 8005 "Sorano" — The Biggest Server CPU Upheaval Since 2019

AMD's EPYC 8005 "Sorano" represents one of the most dramatic generational shifts in server CPU design since the original EPYC launch. The new platform completely reworks AMD's SP6 socket strategy with major architectural changes — new memory controllers, updated cache hierarchy, and I/O fabric redesign. This isn't just a spec bump; it's a fundamental platform re-architecture that will define AMD's server roadmap for years. For bare-metal operators, this means a full platform lifecycle refresh cycle is coming, with implications for cooling, power delivery, and motherboard compatibility.

**Source:** [ServeTheHome](https://www.servethehome.com/amd-epyc-8005-sorano-completely-changes-amd-sp6/)

## 2. 🔥 Lambda Bare Metal Instances — DPU-Powered BMaaS Hits Production

Lambda launched production Bare Metal Instances starting with NVIDIA GB300 NVL72 Superclusters, combining direct hardware access with API-driven lifecycle management. The key innovation: offloading provisioning to NVIDIA BlueField DPUs in zero-trust mode — the DPU handles network identity, NVMe boot device emulation, and storage volume management entirely independent of the host OS. This gives cloud-class programmability (self-service APIs, VPCs, storage volumes, resource governance) on truly unvirtualized hardware. For bare-metal practitioners, this is the "no-compromise" BMaaS pattern — full hardware control with no hypervisor, but cloud-level ops automation.

**Source:** [Lambda Blog](https://lambda.ai/blog/lambda-bare-metal-instances)

## 4. 🔥 Lambda MLPerf Training v6.0: Fastest Llama 3.1 8B on GB300 NVL72

Lambda's MLPerf Training v6.0 submission delivered the fastest Llama 3.1 8B training run on an NVIDIA GB300 NVL72 — an 18.7% improvement over their previous record. Critically, this was the first MLPerf to include MoE (Mixture-of-Experts) benchmarks, and Lambda also submitted the fastest MoE training on NVIDIA HGX B200. MLPerf is increasingly becoming the competitive benchmark for GPU BMaaS providers — Lambda, CoreWeave, Nebius, Oracle, and Dell all submitted GB300 NVL72 results — making this the closest thing to an apples-to-apples BMaaS performance comparison available.

**Source:** [Lambda Blog](https://lambda.ai/blog/mlperf-training-v6.0-lambda-delivers-fastest-llm-training)

## 5. 🔥 NVIDIA CPO Co-Packaged Optics InfiniBand Switch — 3 kW Saved Per Switch

Lambda published a deep-dive unboxing of one of the first NVIDIA Quantum-X InfiniBand Photonics CPO (Co-Packaged Optics) switches, revealing concrete power savings: 3.05 kW saved per switch vs traditional pluggable transceiver-based designs, scaling to ~4.4 MW in a 41,472-GPU cluster. Fewer failure points, less power consumption, and higher reliability — CPO is emerging as a genuine game-changer for GPU cluster fabric. For bare-metal infrastructure teams designing large-scale GPU clusters, CPO eliminates one of the biggest operational pain points: transceiver failure rates in high-density InfiniBand fabrics.

**Source:** [Lambda Blog](https://lambda.ai/blog/unbox-one-of-nvidias-first-co-packaged-optics-samples-with-lambda)

## 6. 🔥 AMD Pivots From HBM to LPDDR5X for Versal Premium Gen 2 — HBM Shortage Hits Server Chip Supply

With HBM in short supply due to GPU demand, AMD's next-generation Versal Premium Gen 2 adaptive SoCs will switch from HBM to LPDDR5X memory-on-package. These chips target a 15+ year projected lifecycle for networking and infrastructure applications. This is a significant bare-metal infrastructure signal — HBM supply constraints are now forcing architectural changes in non-GPU server chips, and LPDDR5X as an HBM alternative could reshape memory subsystem design in future FPGA/adaptive SoC-based accelerators and DPUs.

**Source:** [ServeTheHome](https://www.servethehome.com/amd-pivots-from-hbm-to-lpddr5x-for-new-versal-premium-gen-2-memory-on-package-chips/)

## 8. 🏗️ AI Infrastructure: Scale Up or Scale Out? The Neocloud vs Enterprise Divide

Equinix published a strategic analysis of how different players are scaling AI infrastructure: neoclouds (Lambda, CoreWeave) are primarily scaling up — building denser, more powerful single clusters with faster interconnects — while traditional enterprises are scaling out across colocation footprints for geographic distribution, data sovereignty, and risk mitigation. This divide directly impacts bare-metal procurement strategy — scale-up favors large GPU dense clusters on dedicated hardware, while scale-out favors distributed dedicated servers across colocation hubs. The analysis helps frame whether bare-metal or cloud makes sense for different AI workload categories.

**Source:** [Equinix Blog](https://blog.equinix.com/blog/2026/06/23/ai-infrastructure-scale-up-or-scale-out/)

## 9. 🔬 RISC-V SpacemiT K3 RVV Vector Benchmarks — RISC-V Finally Has Competitive Vector Compute

Phoronix published comprehensive benchmarks of the SpacemiT K3 RISC-V SoC — one of the first commercially available chips supporting the RVA23 profile with RVV 1.0 vector extensions. The vector benchmarks show RISC-V maturing rapidly for compute workloads, with competitive performance in vectorized operations. For bare-metal infrastructure, this matters because RISC-V server CPUs (already appearing in Chinese supercomputing and data centers) need competitive vector/matrix compute to handle AI inference, HPC, and data processing workloads at the server level. This is a foundational benchmark for any bare-metal operator evaluating RISC-V as an alternative to ARM or x86 in the data center.

**Source:** [Phoronix](https://www.phoronix.com/review/risc-v-rvv-vector-benchmarks)

## 10. 🧠 Intel ACE (AI Compute Extensions) — x86 Gets Native AI Instructions for Server CPUs

The x86 Ecosystem Advisory Group, led by Intel and AMD, has firmed up the AI Compute Extensions (ACE) specification for optimizing x86 for AI computation — matrix multiplication, tensor operations, and ML workloads. Intel posted initial GCC compiler patches for ACE, marking the first concrete implementation. This is a long-term bare-metal infrastructure story: native AI instructions in x86 server CPUs mean that general-purpose servers may handle inference, embedding generation, and AI preprocessing without dedicated GPU accelerators, shifting workload placement decisions for bare-metal operators. ACE adds to the growing ecosystem of CPU-based AI compute alongside Intel's AMX and NVIDIA's Grace Hopper.

**Source:** [Phoronix](https://www.phoronix.com/news/Intel-GCC-ACE-AI-Patches)

## 📊 Bare-Metal Trend Summary

- **DPU-powered BMaaS is the new standard** — Lambda's Bare Metal Instances using NVIDIA BlueField DPUs for zero-trust provisioning represent a genuine architectural shift in how bare-metal is delivered. Expect copycat offerings from CoreWeave, Nebius, and traditional dedicated server providers within 6-12 months.
- **HBM shortage is reshaping server chip architecture** — AMD's decision to move Versal SoCs from HBM to LPDDR5X confirms that GPU demand is constraining HBM supply for non-GPU applications. This will drive innovation in memory-on-package alternatives and may accelerate adoption of CXL-based memory pooling in data centers.
- **GPU cluster networking is being reinvented** — NVIDIA CPO switches saving 3 kW each and eliminating transceiver failures shows that networking is no longer a commodity afterthought. Co-packaged optics will become a differentiator for BMaaS providers building large-scale GPU clusters.
- **ARM/RISC-V diversification in the data center is accelerating** — multiple ARM CPU stories (NVIDIA Vera, Qualcomm Dragonfly, AMD Sorano architecture) plus RISC-V RVV benchmarking shows the server CPU market is genuinely diversifying beyond x86. Bare-metal operators should be preparing multi-architecture deployment playbooks.
- **Colocation remains essential for AI infrastructure** — Equinix's HK6 launch and the scale-up vs scale-out analysis confirm that physical colocation is not being displaced by cloud. Enterprises are increasingly using colocation for AI workloads that need hardware control, data sovereignty, or latency guarantees that virtualized cloud can't provide.
