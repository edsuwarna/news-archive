## 🧠 4. Intel & AMD Finalize AI Compute Extensions (ACE) — GCC Compiler Patches Land

Intel engineers have posted initial GCC compiler patches, marking the beginning of software ecosystem enablement. ACE will power the next generation of x86 server CPUs in AI inference and training workloads, a development that directly impacts bare-metal server procurement decisions for AI infrastructure.

## 🌐 7. Lambda Unboxes NVIDIA's First Co-Packaged Optics InfiniBand Switch

Lambda published a deep-dive unboxing of the NVIDIA Quantum-X InfiniBand Photonics Q3450-LD, one of the first co-packaged optics switches to reach a cloud operator. CPO saves 3.05 kW per switch versus traditional pluggable transceivers, scaling to ~4.4 MW in a 41,472-GPU cluster. This is a landmark moment for bare-metal networking: CPO directly improves GPU cluster TCO by cutting the fabric power budget — traditionally 86% of networking power at 800G scale.

**Source:** [Lambda Labs](https://lambda.ai/blog/unbox-one-of-nvidias-first-co-packaged-optics-samples-with-lambda)

## 📡 9. Linux 7.2 Raises Default RISC-V 64-bit CPU Limit to 256 Cores

The Linux kernel has bumped the default maximum supported CPU count for RISC-V 64-bit builds to 256 cores — a major scalability milestone for the open ISA. Combined with the SpacemiT K3 RISC-V SoC benchmarks showing RVA23 profile support and RVV 1.0 vector extension performance, this enables RISC-V to target server-class bare-metal deployments. The combination of 256-core kernel support and competitive vector performance makes RISC-V a progressively more viable option for scale-out bare-metal clusters.

**Source:** [Phoronix](https://www.phoronix.com/news/Linux-7.2-RISC-V-256-Cores)
